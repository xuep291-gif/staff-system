## 六、子角色权限体系设计

### 6.1 角色层级

```
教职工登录
├── 教师端 (type=2)
│   └── 班主任/辅导员 — 本班数据
├── 财务/职工端 (type=3)
│   ├── 收费专员 (subRole: fee_collector)
│   │   └── 权限：线下收款、退费处理、补差退款、票据管理、催缴、打款
│   ├── 审批专员 (subRole: fee_approver)
│   │   └── 权限：助学金审批、助学贷款审批、统计查看
│   └── 迎新工作人员 (subRole: checkin_staff)
│       └── 权限：缴费核验、现场收款登记、报到统计
└── 政务职工端 (type=5)
    ├── 学工处负责人 (subRole: student_affairs)
    │   └── 权限：全校终审、全校换房审批、全校统计
    └── 学院负责人 (subRole: college_dean)
        └── 权限：本院复审、本院统计（只读）
```

### 6.2 权限数据模型

```js
// 登录后后端返回
{
  userId: 'staff_finance_001',
  name: '陈美玲',
  role: 'finance',          // 主角色
  subRole: 'fee_collector', // 子角色
  permissions: [            // 功能权限列表
    'finance:collect',      // 线下收款
    'finance:refund',       // 退费处理
    'finance:diff',         // 补差退款
    'finance:receipt',      // 票据管理
    'finance:urge',         // 催缴任务
    'finance:payout'        // 打款
  ],
  dataScope: {              // 数据范围
    type: 'all',            // 'all' | 'college' | 'class'
    collegeId: null,        // 学院ID（学院负责人时）
    classId: null           // 班级ID（班主任时）
  }
}
```
