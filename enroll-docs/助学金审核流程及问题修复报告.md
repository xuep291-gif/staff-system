## 一、助学金审核流程概览

### 1.1 审核流水线（5 阶段）

```
学生提交申请 → 辅导员初审 → 学院负责人复审 → 学工处审批 → 财务打款 → 完成
     ↓              ↓              ↓              ↓           ↓
  (PENDING)    (FIRST_PASS)   (REVIEW_PASS)   (FINAL_PASS)  (COMPLETED)
```

| 阶段 | 负责角色 | 页面路径 | 入口状态 |
|------|----------|----------|----------|
| 学生提交 | 学生 | — | `pending` |
| 辅导员初审 | 教师端（辅导员） | `teacher/aid-review/index` | `pending` |
| 学院负责人复审 | 政务端（学院） | `government/aid-review/index` | `first_pass` |
| 学工处审批 | 政务端（学工处） | `government/aid-final-review/index` | `review_pass` |
| 财务打款 | 财务端 | `finance/aid-review/index` | `final_pass` / `payment_pending` |

### 1.2 状态定义（`REVIEW_STATUS`）

| 常量 | 值 | 含义 | 下一阶段 |
|------|-----|------|----------|
| `PENDING` | `pending` | 学生已提交，待辅导员初审 | → `FIRST_PASS` 或 `REJECTED` |
| `FIRST_PASS` | `first_pass` | 辅导员初审通过，待学院复审 | → `REVIEW_PASS` 或 `REJECTED` |
| `REVIEW_PASS` | `review_pass` | 学院复审通过，待学工处审批 | → `FINAL_PASS` 或 `REJECTED` |
| `FINAL_PASS` | `final_pass` | 学工处审批通过，待财务打款 | → `PAYMENT_PENDING` 或 `PAID` |
| `PAYMENT_PENDING` | `payment_pending` | 待财务确认打款 | → `PAID` |
| `PAID` | `paid` | 已打款 | → `COMPLETED` |
| `COMPLETED` | `completed` | 流程完成 | — |
| `REJECTED` | `rejected` | 驳回（任一阶段均可） | — |

```
enroll-app/src/pages/
├── teacher/                        # 教师端（辅导员）
│   ├── aid-home/index.vue          #   初审列表页
│   └── aid-review/index.vue        #   初审详情页
├── government/                     # 政务端
```

## 三、遇到的主要问题及解决方案

### 问题 1：学院账号登录后首页显示学工处负责人信息

**现象：**  
学院账号（工号 3002，`subRole: college_dean`）登录后，政务首页 Banner 显示「学工处 · 学工处负责人」，待办事项显示学工处的终审项。

**根因：**  
Mock 服务器 `getPermissionsData()` 函数（`mock/server.js:517`）使用错误的 key 查找用户：

```javascript
// Bug: state.users 的 key 是工号（'3001', '3002'），不是 userId
const user = params.userId ? state.users[params.userId] : null
```

`params.userId` 值是 `'staff_gov_002'`（userId），但 `state.users` 的 key 是 `'3002'`（工号）。查找永远返回 `undefined`，导致函数返回 `fail('用户不存在', 40400)`。连锁反应：
1. `handleAuth` 中 `perm.data` 为 `null` → `subRole = ''`
2. `saveUserInfo` 将空 `subRole` 写入缓存和 `staff_sub_role` storage
3. `getSubRole()` 读不到有效值 → 政务首页回退到 `data()` 默认值（学工处）

**修复：**  
改为遍历 `Object.values(state.users)` 按 `userId` / `workNo` 匹配：

```javascript
function getPermissionsData(state, params) {
  let user = null, accountKey = null
  if (params.userId) {
    for (const [key, u] of Object.entries(state.users)) {
      if (u.userId === params.userId || u.workNo === params.userId) {
        user = u; accountKey = key; break
      }
    }
  }
  if (!user) return fail('用户不存在', 40400)
  // 按工号 → workNo → userId 依次匹配权限
  const perm = permissionsMap[accountKey] || permissionsMap[user.workNo] 
            || permissionsMap[user.userId] || { subRole: '', permissions: [], dataScope: { type: 'all' } }
  return ok({ userId: user.userId, name: user.name, role: user.roles[0], ...perm })
}
```

**涉及文件：** `mock/server.js`

### 问题 2：subRole 持久化链路不可靠

**现象：**  
偶现登录后 `getSubRole()` 返回空字符串，导致政务首页无法区分学工处/学院。

**根因链：**

1. **`$cache` 格式兼容问题** — `$cache` 以 `JSON_|_expireTime` 格式存储，`readUserInfo()` 需要特殊解析该格式
2. **缺少独立存储 key** — `subRole` 仅存在于 `userInfo` 对象内，一旦缓存过期或被清除（如 `App.vue onHide`），`subRole` 丢失
3. **无兜底策略** — 缺少按 `userId` 的硬编码映射作为最后防线

**修复（多道防线）：**

```javascript
// 防线 1：独立 key 存储，绕过 $cache 格式问题
uni.setStorageSync('staff_sub_role', query.subRole || '')
localStorage.setItem('staff_sub_role', query.subRole || '')  // H5 双写

// 防线 2：读取时优先独立 key
export function getSubRole() {
  const direct = uni.getStorageSync('staff_sub_role'); if (direct) return direct
  const user = readUserInfo(); if (user?.subRole) return user.subRole
  // 防线 3：硬编码兜底
  if (user?.userId === 'staff_gov_001') return SUB_ROLES.STUDENT_AFFAIRS
  if (user?.userId === 'staff_gov_002') return SUB_ROLES.COLLEGE_DEAN
  return ''
}
```

**涉及文件：** `utils/permissions.js`, `pages/login/index.vue`

### 问题 3：学院/学工处页面混用，列表数据错乱

**现象：**  
学院负责人和学工处负责人共用同一个助学金列表页，无法区分各自的审核范围。学院复审看到终审项，学工处终审看到初审项。

**修复：**  

1. **拆分独立列表页：**
   - `government/aid-home/index.vue` — 学院复审列表，只显示 `first_pass` 状态
   - `government/aid-final-home/index.vue` — 学工处终审列表，只显示 `review_pass` 状态

2. **拆分独立审核详情页：**
   - `government/aid-review/index.vue` — 学院复审，`canReview` 判断 `expectedStatus === 'first_pass'`
   - `government/aid-final-review/index.vue` — 学工处审批，`canReview` 判断 `expectedStatus === 'review_pass'`

3. **路由分发：** 政务首页 `goTodo('aid-home')` 根据 `isSchool` 路由到正确页面：
   ```javascript
   'aid-home': this.isSchool 
     ? '/pages/government/aid-final-home/index'   // 学工处 → 终审
     : '/pages/government/aid-home/index'          // 学院 → 复审
   ```

**涉及文件：** `government/aid-home/index.vue`, `government/aid-final-home/index.vue`, `government/aid-review/index.vue`, `government/aid-final-review/index.vue`, `government/home/index.vue`

### 问题 4：审核页误显示操作按钮（状态校验不严）

**现象：**  
某些处于非当前阶段（如已完结/审批中的后续阶段）的申请，在审核页仍显示「通过/驳回」按钮，但操作后状态跳转混乱。

**根因：**  
`canReview` 判断仅检查 `item.status` 是否为某个硬编码值，未与**列表页传入的预期状态**对比。当用户直接输入 URL 访问审核页时，可能绕过列表页的状态过滤。

**修复：**

1. **列表页传 `status` 参数：**
   ```javascript
   uni.navigateTo({ url: `/pages/government/aid-review/index?uid=${item.uid}&status=${item.status}` })
   ```

2. **审核页按 `expectedStatus` 判断可审：**
   ```javascript
   // 学院复审页 — 只有 first_pass 状态才可操作
   canReview() {
     return this.expectedStatus === REVIEW_STATUS.FIRST_PASS
   }
   // 学工处审批页 — 只有 review_pass 状态才可操作
   canReview() {
     return this.expectedStatus === REVIEW_STATUS.REVIEW_PASS
   }
   ```

3. **辅导员初审页增加入口守卫：** 非 `pending` 状态直接 toast 提示并返回：
   ```javascript
   if (this.item.status !== REVIEW_STATUS.PENDING) {
     uni.showToast({ title: '该申请不在辅导员初审阶段', icon: 'none' })
     setTimeout(() => uni.navigateBack(), 800)
     return
   }
   ```

**涉及文件：** `teacher/aid-review/index.vue`, `government/aid-review/index.vue`, `government/aid-final-review/index.vue`, `government/aid-home/index.vue`, `government/aid-final-home/index.vue`

## 五、审核操作流程（端到端）

### 5.1 辅导员初审（教师端）

```
1. 教师端首页 → 助学金待办 → aid-home 列表页
2. 列表显示所有 pending 状态的申请
3. 点击进入 aid-review 详情页
4. 可修改助学金类型（自研下拉选择器）
5. 填写审核意见 → 点击「初审通过」
6. 状态更新为 first_pass → 进入学院复审池
```

### 5.2 学院负责人复审（政务端-学院）

```
1. 政务端首页（学院） → 助学金复审待办 → aid-home 列表页
2. 列表只显示 first_pass 状态的申请
3. 点击进入 aid-review 详情页
4. 填写复审意见 → 点击「复审通过」
5. 状态更新为 review_pass → 进入学工处审批池
```

### 5.3 学工处审批（政务端-学工处）

```
1. 政务端首页（学工处） → 助学金终审待办 → aid-final-home 列表页
2. 列表只显示 review_pass 状态的申请
3. 点击进入 aid-final-review 详情页
4. 填写审批意见 → 点击「审批通过」
5. 状态更新为 final_pass → 进入财务打款池
```

### 5.4 财务打款（财务端）

```
1. 财务端首页 → 助学金 → aid-review 页面
2. 确认打款金额 → 点击「确认打款」
3. 二次确认弹窗 → 确认
4. 状态更新为 paid/completed
```
