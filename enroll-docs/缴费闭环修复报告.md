## 修改清单

### 修改 1：Java 后端 — savePaymentRecord 增加 billing 表同步（P0 根因修复）

**文件**：[BillController.java](d:\study\backend\enroll-service-java\src\main\java\com\jfeat\enroll\controller\student\BillController.java#L115-L195)

**问题**：学生支付后只写 `t_data_fee_payment_record`，不更新 `t_data_fee_billing`，导致教师端永远显示 paidAmount=0。

**修改内容**：
1. INSERT payment_record 之后，解析 `payItems`（如"学费 + 教材费"），按费项匹配 billing 行
2. `UPDATE t_data_fee_billing SET paid_amount = paid_amount + ?, status = CASE WHEN ... THEN 'PAID'/'PARTIAL' END`
3. 若 billing 为空（首次缴费），先从 `t_data_fee_item` 表读取费项配置，自动初始化 5 行 billing（学费/教材费/住宿费/体检费/军训服装费），每行带 `fee_type` 和 `fee_name`
4. 按费项分配支付金额（如支付"学费 + 教材费"，则分别更新学费行和教材费行的 paid_amount）

### 修改 2：Java 后端 — billList 初始化 billing + 返回 feeType/feeName

**文件**：[BillController.java](d:\study\backend\enroll-service-java\src\main\java\com\jfeat\enroll\controller\student\BillController.java#L30-L70)

4. 增加 `paidAmount` 字段（保留原 `paid` 布尔字段兼容）

## 修改后数据流

```
学生支付 → Java savePaymentRecord
  ├─ INSERT t_data_fee_payment_record（流水）
  ├─ 初始化 billing 行（首次，5 个费项，带 fee_type/fee_name）
  └─ UPDATE t_data_fee_billing.paid_amount, status  ← 新增

教师 fee-home(onShow) → refresh() → GET /payments/students
  └─ SUM(b.paid_amount) → 包含最新支付金额 ✓

教师 student-detail(onLoad) → GET /payments/students/:id
  └─ bills[].itemName → "学费" / "住宿费" / ... ✓
  └─ bills[].paidAmount → 已更新 ✓

财务 home(onShow) → await getFeeList() → GET /payments/students
  └─ 真实 API 数据 ✓

财务 verify(searchStudent) → await getFeeList() → 真实 API 数据 ✓
```
