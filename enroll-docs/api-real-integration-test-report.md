## 当前阻塞

| 编号 | 问题 | 影响 | 建议 |
|---|---|---|---|
| 1 | 配置的真实数据库健康检查返回 `503 database disconnected`。 | 真实账号登录、个人资料更新及数据库持久化数据未能完成联机验收。 | 在可访问数据库的网络环境复测 `/health` 与三角色账号登录，并提供联调用测试账号。 |
| 2 | 后端不存在 `/api/v1/bill/*` 路由。 | `billApi` 尚不能切换到真实接口。 | 后端补账单路由，或前后端确认以 `/payments/*` 代替。 |
| 3 | 后端不存在 `/api/v1/config/*` 路由。 | 收费配置类 service 无真实实现。 | 后端补收费项目/标准配置接口。 |
| 4 | 后端不存在 `/api/v1/dormitory/stats` 与 `/api/v1/dormitory/assign`。 | 住宿统计/分配相关页面尚无法完整真实化。 | 以现有 `/dormitories/*` 扩展统计和分配路由。 |
| 5 | 后端不存在 `/api/v1/student/onsite/payment`。 | 现场登记缴费方法无法调用。 | 确认合并到线下收款确认流程，或新增接口。 |
| 6 | 住宿申请、军训尺码等部分页面仍使用本地演示状态。 | 这些次级页面尚非端到端真实联调状态。 | 在缺失路由补齐后继续逐页迁移至现有 service。 |

## 复测入口

1. 后端：在 `enroll-js` 运行 `npm run build` 后启动 `node dist/index.js`。
2. 前端：在 `enroll-app` 运行 `npm run build:h5` 或开发服务。
3. 默认真实 API：`http://localhost:3100`。
4. 临时切回 mock：设置 `VITE_USE_MOCK=true` 后重新启动前端。
